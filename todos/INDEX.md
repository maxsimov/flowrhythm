## Status values

- **planned** — items identified, no work started
- **in-progress** — at least one item being worked on
- **implemented** — all items done; file kept as a record

## How to update

- When starting work on a plan, set status to `in-progress` in the file's header AND update the index
- Mark individual items `- [x]` as they're done; add a brief note about how/where
- When all items are done, set status to `implemented` and update the index
- Never delete a file — it's a record of what was decided and shipped
